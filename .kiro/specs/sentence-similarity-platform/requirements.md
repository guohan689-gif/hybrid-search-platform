# 需求文档

## 简介

句子向量相关性分析平台是一个基于Vue.js的Web应用，允许用户输入多个句子，系统将计算这些句子的向量表示，分析它们之间的相似度，并提供相似度高或低的可能原因解释。该平台旨在帮助用户理解文本语义相似性，适用于内容分析、文本比对、语义搜索等场景。

## 术语表

- **系统（System）**: 句子向量相关性分析平台
- **句子向量（Sentence_Vector）**: 句子的数值化表示，通常是高维向量
- **相似度（Similarity）**: 两个句子向量之间的相关性度量，范围通常在0到1之间
- **向量化引擎（Vectorization_Engine）**: 负责将文本句子转换为向量表示的组件
- **相似度计算器（Similarity_Calculator）**: 负责计算向量之间相似度的组件
- **原因分析器（Reason_Analyzer）**: 负责分析并解释相似度结果的组件
- **用户界面（UI）**: 用户与系统交互的Vue.js前端界面

## 需求

### 需求 1: 句子输入

**用户故事:** 作为用户，我想要输入多个句子，以便系统可以分析它们之间的相似度。

#### 验收标准

1. 当用户访问平台时，系统应显示句子输入界面
2. 当用户输入句子时，系统应支持至少2个句子的输入
3. 当用户输入句子时，系统应支持最多10个句子的输入
4. 当用户输入空白句子时，系统应拒绝该输入并提示用户
5. 当用户输入的句子超过1000个字符时，系统应拒绝该输入并提示用户

### 需求 2: 句子向量化

**用户故事:** 作为系统，我需要将输入的句子转换为向量表示，以便进行相似度计算。

#### 验收标准

1. 当用户提交有效句子时，向量化引擎应将每个句子转换为句子向量
2. 当向量化过程失败时，系统应返回错误信息并通知用户
3. 当句子包含特殊字符或多语言文本时，向量化引擎应正确处理
4. 向量化引擎应使用预训练的语言模型（如Sentence-BERT或类似模型）

### 需求 3: 相似度计算

**用户故事:** 作为系统，我需要计算句子向量之间的相似度，以便向用户展示结果。

#### 验收标准

1. 当句子向量生成完成时，相似度计算器应计算所有句子对之间的相似度
2. 当计算相似度时，系统应使用余弦相似度作为度量标准
3. 当相似度计算完成时，系统应返回范围在-1到1之间的相似度分数
4. 当有N个句子时，系统应生成N×(N-1)/2个相似度对

### 需求 4: 结果展示

**用户故事:** 作为用户，我想要看到清晰的相似度结果展示，以便理解句子之间的关系。

#### 验收标准

1. 当相似度计算完成时，用户界面应以矩阵形式展示所有句子对的相似度
2. 当展示相似度时，系统应使用颜色编码表示相似度高低（如热力图）
3. 当用户点击某个相似度结果时，系统应高亮显示对应的句子对
4. 当展示结果时，系统应显示相似度的数值（保留2位小数）
5. 用户界面应提供清晰的视觉反馈，使用户能够快速识别高相似度和低相似度的句子对

### 需求 5: 原因分析

**用户故事:** 作为用户，我想要了解句子相似度高或低的原因，以便更好地理解分析结果。

#### 验收标准

1. 当用户选择某个句子对时，原因分析器应分析并展示相似度的可能原因
2. 当相似度高时，系统应识别并展示共同的关键词、主题或语义特征
3. 当相似度低时，系统应识别并展示不同的主题、词汇差异或语义差异
4. 当展示原因时，系统应以列表形式呈现至少3个主要原因
5. 原因分析器应提取并高亮显示句子中的关键词或短语

### 需求 6: 数据持久化

**用户故事:** 作为用户，我想要保存我的分析历史，以便日后查看或比较。

#### 验收标准

1. 当用户完成一次分析时，系统应将输入句子和结果保存到本地存储
2. 当用户重新访问平台时，系统应显示最近的分析历史（最多保存10条）
3. 当用户选择历史记录时，系统应加载并展示该次分析的完整结果
4. 当用户删除历史记录时，系统应从本地存储中移除对应数据

### 需求 7: 错误处理

**用户故事:** 作为用户，我希望在出现错误时能够收到清晰的提示，以便知道如何解决问题。

#### 验收标准

1. 当向量化失败时，系统应显示友好的错误消息并建议用户重试
2. 当网络请求失败时，系统应显示网络错误提示
3. 当输入验证失败时，系统应在输入框旁边显示具体的验证错误信息
4. 当系统遇到未预期的错误时，系统应显示通用错误消息并记录错误日志

### 需求 8: 性能要求

**用户故事:** 作为用户，我希望系统能够快速响应，以便获得流畅的使用体验。

#### 验收标准

1. 当用户提交2-5个句子时，系统应在3秒内完成向量化和相似度计算
2. 当用户提交6-10个句子时，系统应在10秒内完成向量化和相似度计算
3. 当计算进行时，用户界面应显示加载指示器和进度信息
4. 用户界面应在用户交互后100毫秒内提供视觉反馈

### 需求 9: 响应式设计

**用户故事:** 作为用户，我希望能够在不同设备上使用该平台，以便随时随地进行分析。

#### 验收标准

1. 当用户在桌面浏览器访问时，用户界面应充分利用屏幕空间
2. 当用户在平板设备访问时，用户界面应自动调整布局以适应屏幕尺寸
3. 当用户在移动设备访问时，用户界面应提供简化的单列布局
4. 当屏幕尺寸改变时，用户界面应平滑过渡到相应的布局

### 需求 10: 导出功能

**用户故事:** 作为用户，我想要导出分析结果，以便在其他地方使用或分享。

#### 验收标准

1. 当用户完成分析时，系统应提供导出按钮
2. 当用户点击导出时，系统应生成包含所有句子和相似度结果的JSON文件
3. 当用户点击导出时，系统应提供CSV格式选项，包含句子对和相似度分数
4. 当导出完成时，系统应自动下载文件到用户设备
